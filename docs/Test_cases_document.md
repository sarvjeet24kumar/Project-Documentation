| **Test Case No.** | **Module Name**                                    | **Test Case Description**                                                                  | **Acceptance Criteria**                                                                                  |
| ----------------- | -------------------------------------------------- | ------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------- |
| **T.C. 1**        | **User Authentication - Signup**                   | Verify that a new user can successfully register with valid username, email, and password. | The system should create a new user record, return a 201 status code, and confirm success message.       |
| **T.C. 2**        | **User Authentication - Signup (Duplicate Email)** | Attempt to register a user with an existing email.                                         | The API should return a 400 status code with an appropriate error message like *“Email already exists”*. |
| **T.C. 3**        | **User Authentication - Login**                    | Verify that a registered user can log in with correct credentials.                         | The system should authenticate successfully, return a valid token (JWT/session), and 200 status code.    |
| **T.C. 4**        | **User Authentication - Invalid Login**            | Attempt login with incorrect password.                                                     | The API should return a 401 Unauthorized error with message *“Invalid credentials”*.                     |
| **T.C. 5**        | **Admin - Create Event (Valid Data)**              | Verify Admin can create events with valid title, description, date, and capacity.          | Event should be created, stored in database, and return 201 Created.                                     |
| **T.C. 6**        | **Admin - Create Event (Conflict Validation)**     | Try creating two events with overlapping time slots.                                       | API should prevent duplicate/conflicting events and return 400 with *“Time conflict detected”*.          |
| **T.C. 7**        | **Student - Register for Event**                   | Verify student can register for available event.                                           | Registration record should be created, seat count should decrease by one, and return 201 Created.        |
| **T.C. 8**        | **Student - Register (Seat Full)**                 | Try to register when event seat count is 0.                                                | API should return 400 error with message *“No seats available”*.                                         |
| **T.C. 9**        | **Student - Concurrency Handling**                 | Simulate multiple students registering simultaneously for limited seats.                   | Only available number of seats should be booked; no overbooking should occur.                            |
| **T.C. 10**       | **Student - Unregister Event**                     | Verify student can unregister from event before it starts.                                 | Registration should be deleted, and seat count should increase by one.                                   |
| **T.C. 11**       | **Admin - Delete Event (Cascade Check)**           | Delete event and verify related student registrations are removed automatically.           | Event and all related registrations should be deleted (cascade delete).                                  |
| **T.C. 12**       | **Security - Unauthorized Event Deletion**         | Attempt to delete event by a Student user.                                                 | API should return 403 Forbidden with message *“Permission denied”*.                                      |
| **T.C. 13**       | **API Response Validation**                        | Verify that all API endpoints return correct HTTP status codes and JSON responses.         | Responses must follow REST conventions with proper codes (200, 201, 400, 401, 403, 404).                 |
